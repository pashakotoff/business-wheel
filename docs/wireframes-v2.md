# Вайрфреймы v2: Двухколоночная версия — Колесо Баланса Бизнеса

## Обзор концепции

**Версия:** `index-v2.html`
**Layout:** Двухколоночный split-screen, 55% / 45%
**Ключевая идея:** Правая панель — «умный помощник», контекст и мотивация в реальном времени. Левая — фокус на действии.

## 5. UX-рекомендации

### 5.1 Фиксация левой панели в viewport

Левая панель — строго `height: 100vh; overflow: hidden`. Вопрос всегда в центре экрана. Это критически важно: пользователь не должен скроллить, чтобы увидеть кнопки 1–10.

Центрирование вопроса по вертикали:
```css
.panel-left {
  display: flex;
  flex-direction: column;
  justify-content: center;
  padding: 40px 48px;
}
```

### 5.2 Анимации переходов между секторами

**Переход между вопросами внутри сектора:**
- Fade out текущего вопроса: `opacity 1→0, translateX 0→-20px`, 200ms
- Fade in нового: `opacity 0→1, translateX 20px→0`, 250ms
- Задержка между out/in: 50ms

**Переход между секторами (смена всей левой панели):**
- Левая панель: slide out влево `translateX 0→-100%`, 350ms ease-in
- Правая панель: cross-fade описания сектора, 400ms
- Живое колесо: плавная заливка нового сектора реальным цветом, 600ms ease-out
- Левая панель: slide in справа `translateX 100%→0`, 350ms ease-out

**Первое появление колеса (landing → quiz start):**
- Колесо «разворачивается» — strokeDashoffset-анимация, 800ms
- Все секторы пустые (пунктир), текущий — пульсирует

**Заполнение сектора колеса:**
```
Событие: пользователь ответил на 4-й вопрос сектора
→ Рассчитать среднее
→ Анимировать заливку сектора: opacity 0→1 + scale 0.8→1, 500ms ease-out
→ Цвет = danger/warning/success по результату
→ Мини-вибрация колеса: scale 1→1.03→1, 200ms
```

**Разблюривание колеса на форме контактов:**
```
onInput(name):  blur 14px → 10px, transition 600ms
onInput(phone): blur 10px → 7px,  transition 600ms
onInput(email): blur 7px  → 4px,  transition 600ms
onSubmit:       blur 4px  → 0px + opacity overlay 0.7→0, 800ms
```

### 5.3 Состояния шкалы оценок

Кнопки 1–10: при выборе значения — немедленный visual feedback:
- Кнопка окрашивается в цвет зоны (1–4: danger, 5–7: warning, 8–10: success)
- Кнопки ниже выбранного — тонкая подсветка в том же цвете, opacity 0.3
- Кнопки выше — нейтральны

Это даёт пользователю мгновенное понимание «что значит моя оценка» без объяснений.

### 5.4 Прогресс-бар

- Два уровня прогресса: глобальный (секторы 1–8) + локальный (вопросы 1–4 внутри сектора)
- Глобальный: полоса вверху, заполняется по 12.5% за сектор
- Локальный: точки или засечки над числом вопроса
- Цвет прогресс-бара: градиент amber → blue (0%→100%)

### 5.5 Выбор типа бизнеса

Две карточки занимают всю ширину левой панели:
```
+------------------------------+ +------------------------------+
|                              | |                              |
|  [Package icon]              | |  [Star icon]                 |
|                              | |                              |
|  ТОВАРЫ                      | |  УСЛУГИ                      |
|                              | |                              |
|  Производство, торговля,     | |  Консалтинг, агентства,      |
|  склад, физический продукт   | |  обучение, сервис            |
|                              | |                              |
+------------------------------+ +------------------------------+
```
- Hover: border amber + translateY(-4px)
- Клик: немедленно активирует карточку + переход к сектору 7

### 5.6 Микро-фидбек при переходе между секторами

Вместо отдельного экрана микро-фидбека (как в v1) — в v2 это решается через правую панель:
- При завершении сектора: плашка в правой панели над колесом на 2 секунды
- «Стратегия: 7.5/10» + цвет зоны + icon checkmark
- Затем автоматически переходит к следующему сектору

### 5.7 Sticky header квиза

На десктопе прогресс-бар + название сектора — sticky сверху левой панели:
```css
.quiz-header {
  position: sticky;
  top: 0;
  z-index: 10;
  background: var(--color-bg);
  padding: 16px 48px;
  border-bottom: 1px solid var(--color-border);
}
```

### 5.8 Состояние результатов — последовательная анимация

Элементы результатов появляются последовательно, не одновременно:

```
0ms    — Заголовок «Ваш индекс системности» fade in
200ms  — Число балла count-up анимация (0 → 62), 800ms
1000ms — Статус-строка slide up
1200ms — Кнопка CTA fade in
───────────────────────────────── (правая панель)
0ms    — Колесо начинает прорисовываться (strokeDashoffset)
800ms  — Заливка секторов цветом, один за одним, 100ms интервал
1600ms — Легенда fade in
1800ms — Таблица секторов появляется построчно
2200ms — Карточки слабых зон slide up
2600ms — CTA кнопка в правой панели
```

### 5.9 Доступность (WCAG 2.1 AA)

- Все интерактивные элементы: `min-height: 44px, min-width: 44px`
- Focus ring: `outline: 2px solid #F59E0B; outline-offset: 3px`
- Кнопки 1–10: `aria-label="Оценка X"` + `role="radio"` в группе
- Живое колесо: `aria-live="polite"` на блок прогресса, обновляется при смене сектора
- Blurred wheel: `aria-hidden="true"` (декоративный элемент)
- Переходы: `prefers-reduced-motion: reduce` — все анимации → instant

### 5.10 Производительность

- Правая панель: колесо рендерится как инлайновый SVG, не Canvas (лучше для accessibility и анимаций CSS)
- Описания секторов: все 8 загружаются в DOM сразу, переключаются через CSS `display: none / block` — нет задержек
- Колесо landing: статичный SVG-файл (не генерируется JS)
- Живое колесо: генерируется один раз, обновляется через изменение атрибутов SVG `<path d="">` — не перерисовка DOM

## 6. Сводная схема состояний правой панели

```
STATE MACHINE — ПРАВАЯ ПАНЕЛЬ

[LANDING]
  └─ Статичное колесо + соцдоказательство + цитата

    ↓ клик «Пройти тест»

[QUIZ: sector N, question M]
  ├─ Зона A: Иконка + название + описание сектора N
  ├─ Зона B: Живое колесо (секторы 1..N-1 залиты, N — пульсирует, N+1..8 — пунктир)
  └─ Зона C: Мотивационный текст «N из 8 заполнено»

    ↓ ответ на вопрос 4 сектора N
    → Сектор N заливается цветом
    → Плашка: «[Сектор]: [score]» на 2с
    → Переход к сектору N+1 (или к выбору типа если N=6)

[BUSINESS TYPE CHOICE]
  ├─ Пояснение «Зачем нам это знать»
  ├─ Колесо: 6 залиты, 7 — пунктир пульсирует
  └─ Подсказка по выбору

    ↓ выбор типа

[QUIZ: sector 7 (выбранный тип)]
  └─ То же, что Quiz, но с контентом под выбранный тип

[CONTACT FORM]
  ├─ Blurred колесо (все 8 секторов, blur 14px)
  ├─ «Что вы получите» — 4 пункта
  └─ Гарантия конфиденциальности

    ↓ onInput → blur уменьшается поэтапно
    ↓ submit

[RESULTS]
  ├─ Полное цветное колесо (анимированная прорисовка)
  ├─ Детальная таблица 8 секторов с оценками
  ├─ 2 карточки слабых зон с разбором
  └─ CTA «Разобрать с экспертом»
```
