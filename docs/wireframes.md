# Вайрфреймы: Тест на системность бизнеса — Колесо Баланса Бизнеса

## Информационная архитектура

### Цели пользователя:
1. Быстро понять, о чём тест, и принять решение пройти его (лендинг)
2. Честно ответить на 32 вопроса и дойти до конца без потери мотивации
3. Получить персональный результат — наглядную карту слабых зон бизнеса
4. Принять решение: записаться на диагностику или сохранить/поделиться результатом

### Приоритет контента (от важного к менее важному):
1. Заголовок с конкретной выгодой — «Узнайте, что тормозит бизнес за 3 минуты»
2. Визуал колеса баланса — сразу показывает формат результата
3. Кнопка «Пройти тест бесплатно» — единственное целевое действие на лендинге
4. Микродоверие — счётчик прошедших, имя и фото автора
5. Прогресс-бар теста — снижает тревогу незавершённости
6. Вопросы теста — чёткие, по одному блоку за раз
7. Форма контактов — мотивирована размытым превью результата
8. Radar chart с результатом — главная «награда» за прохождение
9. Расшифровка слабых секторов — персонализированный инсайт
10. CTA на диагностику — конвертация в клиента

```

         │ клик «Пройти тест»
         ▼

[Экран 9] Сектор 7: Производство (Товары) / Услуги
```

**Поведение на мобильном:**
- Вопросы отображаются по одному (accordion-стиль): после ответа на вопрос N, вопрос N+1 появляется через плавный scroll-reveal (slide-down + fade-in, 300ms)
- После ответа на вопрос 4 — кнопка «Далее» становится активной
- Нижняя навигация (Назад / Далее) зафиксирована внизу экрана (sticky, 64px)

**Поведение на десктопе:**
- Все 4 вопроса видны одновременно, отвечать можно в любом порядке
- Кнопка «Следующий сектор» — справа внизу блока
- После клика на кнопку оценки: кнопка подсвечивается акцентным цветом, микроанимация «pulse» (scale 1.1 → 1, 150ms)

**Микро-фидбэк между секторами:**
После нажатия «Следующий сектор»: появляется toast/overlay на 1.5s — «Готово! Переходим к финансам» (или следующему сектору). Overlay тёмный, полупрозрачный, текст белый по центру. Затем экран следующего сектора slide-in справа налево.

## Навигация

### Десктоп
- Тип: отсутствует как таковая (SPA без традиционного меню — это инструмент, не сайт)
- Логотип «Business Commandos» — в шапке лендинга (статичный, не кликабельный внутри теста)
- Внутри теста: только прогресс-бар и кнопки «Назад» / «Далее»
- На экране результатов: логотип вверху страницы как якорь бренда

### Мобильная
- Нет бургер-меню (SPA-инструмент без навигации)
- Навигация внутри теста: sticky bottom bar с кнопками «Назад» / «Далее» (64px, тёмный фон, border-top)
- Back-кнопка браузера: обрабатывается JS (history.pushState) — возврат к предыдущему сектору

```
Active:    [Следующий сектор →]  → opacity: 1.0, cursor: pointer, акцентный фон
```

## Доступность (Accessibility)

- **Skip-to-content**: скрытая ссылка в начале страницы, видна при фокусе Tab, ведёт к `#main-content`
- **Порядок фокуса**: логичный сверху вниз — логотип → заголовок → кнопка CTA (лендинг); прогресс-бар → заголовок сектора → вопрос 1 → кнопки 1-10 → вопрос 2 → ... → Назад → Далее (тест)
- **ARIA-landmarks**:
  - `<main>` — основной контент
  - `role="progressbar"` с `aria-valuenow`, `aria-valuemin`, `aria-valuemax` для прогресс-бара
  - `role="group"` с `aria-labelledby` для каждого вопроса и его кнопок оценок
  - `aria-label` на каждой кнопке оценки: «Оценка 7 из 10»
  - `aria-disabled="true"` для неактивной кнопки «Далее»
  - `aria-live="polite"` для сообщений об ошибках формы
  - `aria-label` на SVG колесе: «Radar chart: результаты теста по 8 секторам»
- **Все изображения**: alt-тексты (фото Павла: «Павел Котов, эксперт по систематизации бизнеса»)
- **Формы**: `<label>` для каждого `<input>`, связаны через `for`/`id`
- **Контрастность**: минимум 4.5:1 для текста (белый на тёмном фоне — ~15:1, серый #A0AEC0 на #0D1B2A — ~5:1)
- **Клавиатурная навигация**: все кнопки оценок доступны через Tab, выбор через Space/Enter, стрелками между кнопками в ряду (роль `radiogroup`)
- **Фокус-индикатор**: outline 2px акцентного цвета + offset 2px для всех интерактивных элементов
