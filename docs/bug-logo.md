# Bug: Логотип отображается слишком маленьким
**Проект:** business-wheel
**URL:** http://audit.businesscommandos.ru
**Статус:** ✅ РЕШЁН (2026-03-04)

---

## Описание проблемы
Логотип в шапке сайта отображался визуально очень маленьким (~13–20px) несмотря на то, что CSS прописывал height: 72px. В DevTools вкладка Styles показывала правильные правила, но визуальный результат не соответствовал.

## Контекст
Оригинальный дизайн использовал текстовый логотип:
```html
<span class="logo">BC</span>
```
CSS `.logo` был заточен под текст (font-family, font-size). Позже в шаблон вставили PNG-картинку вместо текста, но CSS изначально под img не проектировался. Это и стало корнем всех конфликтов.

---

## КОРНЕВАЯ ПРИЧИНА (найдена 2026-03-04)

**В git был закоммичен неправильный файл изображения с первого же коммита (874be03, 2026-03-03).**

- Git (deployed): `logo-horizontal-white.png` = **1000×1000px** квадрат, 13025 байт
- Диск (локально): `logo-horizontal-white.png` = **516×208px** горизонтальный логотип, 6743 байт

Правильный файл существовал только на локальном диске и **никогда не был закоммичен в git**. GitHub Pages деплоил из git — значит на продакшн всегда уходил квадратный 1000×1000 файл.

Это объясняло все симптомы:
- `height: 72px; width: auto` на квадрате 1000×1000 = 72×72px (квадратный маленький значок вместо горизонтального логотипа 179×72)
- `height: auto` из глобального `img {}` мог дополнительно мешать через специфичность
- Файлы HTML и CSS были "идентичны" — потому что проблема была в самом PNG

Дополнительные факторы:
- `.nojekyll` файл отсутствовал в `src/` (хотя в данном случае не был причиной бага)
- Фраза из лога "516×208px — не причина" была ошибочной: правильный размер был только локально, а deployed — квадрат

---

## Факты (подтверждены)
- PNG файл (локальный): 516×208px, правильный горизонтальный логотип
- PNG файл (git/deployed ДО фикса): 1000×1000px, квадратный значок — НЕПРАВИЛЬНЫЙ
- HTML и CSS на localhost и deployed — идентичны (но сравнивали не PNG!)
- DevTools Styles показывает `height: 72px` как применённое правило
- Deployed CSS проверен через curl — содержимое правильное
- JS (main.js) логотип не трогает
- Service Worker — не зарегистрирован
- transform/scale на родительских элементах — нет
- Браузерные расширения — исключены (проверено в инкогнито, баг воспроизводится)
- CDN кэш — не виновен (проверено через Cache-Control headers)

## Хронология попыток

### ❌ Попытка 1 — убрать inline атрибут
**Что:** Убрали `height="36"` с тега `<img>`
**Гипотеза:** inline атрибут переопределял CSS
**Результат:** Логотип всё равно маленький

### ❌ Попытка 2 — перебор значений height
**Что:** CSS height: 52 → 72 → 88 → 80 → 72px
**Гипотеза:** неправильный размер
**Результат:** Не помогло, размер не менялся визуально

### ❌ Попытка 3 — !important
**Что:** `height: 72px !important; width: auto !important`
**Гипотеза:** что-то переопределяет CSS с большей специфичностью
**Результат:** DevTools показывает !important, визуально без изменений

### ❌ Попытка 4 — явная ширина
**Что:** `width: 179px !important; max-width: none !important`
**Гипотеза:** ширина не вычисляется корректно
**Результат:** Логотип растянулся по пропорциям неправильно, высота осталась малой

### ❌ Попытка 5 — div-обёртка + height: 100%
**Что:** `<div class="logo"><img></div>`, div с `height: 72px`, img с `height: 100%`
**Гипотеза:** img нужен контейнер с явной высотой
**Результат:** Картинка стала квадратом 72×72 (баг с width: auto + max-width: 100%)

### ❌ Попытка 6 — div-обёртка + max-width: none
**Что:** Добавили `max-width: none` на `.logo img`
**Гипотеза:** глобальный `img { max-width: 100% }` создаёт циклическую зависимость
**Результат:** Квадрат исчез, но логотип всё равно маленький

### ❌ Попытка 7 — убрать div, img напрямую в flex
**Что:** `<img class="site-logo">` без обёртки, прямой flex-item
```css
.site-logo { height: 72px; width: auto; max-width: none; display: block; flex-shrink: 0; }
```
**Гипотеза:** div-обёртка с унаследованным CSS от текстового `.logo` создавала конфликт
**Результат:** Задеплоено (v=9). Баг остался — правильный PNG всё ещё не в git.

## ✅ Попытка 8 — ИСПРАВЛЕНИЕ КОРНЕВОЙ ПРИЧИНЫ (2026-03-04)

**Что сделано:**

1. **Обнаружено расхождение PNG**: через `curl -sI` выяснили что deployed PNG = 13025 байт vs локальный = 6743 байт. Через `python3` проверили dimensions: deployed = 1000×1000px, локальный = 516×208px.

2. **Заменили неправильный PNG**: закоммичен правильный 516×208px горизонтальный логотип (был только на диске, никогда не попадал в git).

3. **Переключили лого на CSS background-image**: убрали `<img>` из HTML, заменили на `<div class="site-logo" role="img" aria-label="Business Commandos"></div>`. В CSS:
```css
.site-logo {
  width: 179px;
  height: 72px;
  background: url('../images/logos/logo-horizontal-white.png') no-repeat left center / contain;
  flex-shrink: 0;
}
```
Это полностью обходит конфликт с глобальным `img { height: auto; max-width: 100% }`.

4. **Добавили `.nojekyll`** в `src/` для корректной работы GitHub Pages.

5. **Bump CSS до v=11**.

**Коммит:** `d22c6ac`
**Deploy:** GitHub Actions run успешно завершён
**Верификация:** curl подтвердил — deployed PNG = 6743 байт = 516×208px ✅

## Урок

Всегда проверять **не только HTML/CSS, но и бинарные файлы** (изображения) на расхождение между локальным диском и git. В данном случае `git diff --stat` показал бы расхождение PNG сразу — но исторически сравнивали только текстовые файлы.

Команда диагностики:
```bash
# Размер deployed файла
curl -sI "https://domain.com/images/logo.png" | grep content-length

# Dimensions через python3
python3 -c "
import struct
with open('logo.png', 'rb') as f: data = f.read()
w = struct.unpack('>I', data[16:20])[0]
h = struct.unpack('>I', data[20:24])[0]
print(f'{w}x{h}')
"
```
